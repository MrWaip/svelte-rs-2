# Porting from Svelte compiler

Reference Svelte compiler source is in `reference/compiler/`.

**Use reference to understand WHAT, not HOW.**

- Match the JS output exactly — it defines correctness.
- Do not copy reference naming, file layout, control flow, or data structures.
- Do not port code line-by-line. Translate intent.

**We write Rust from scratch.** That means:

- Shape the code for Rust ergonomics: ownership, enums over polymorphism, explicit errors over panics.
- Prefer flat, explicit logic over the visitor-pattern indirection used in reference.
- Collapse reference's accidental complexity — multiple passes, helper closures, implicit state mutation — into one pass where possible.
- Name things for the code we actually have, not for the JS analogue.

**Optimize for maintainability and performance.** Reference picked JS-friendly patterns (object spreads, ad-hoc `state` mutation, nested visitors); we pick Rust-friendly ones (borrowed `&mut State`, typed buckets, early returns, monomorphic dispatch).

Never edit by hand case-_.json, case-_.js files. Only generated by `just generate`.
