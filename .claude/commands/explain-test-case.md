# Explain test case: $ARGUMENTS

Read-only analysis of an existing compiler test case. Never edit any files.

## Step 1: Validate

If `$ARGUMENTS` is empty, ask the user for a test case name and stop.

Check that `tasks/compiler_tests/cases2/$ARGUMENTS/` exists. If not, report and stop.

## Step 2: Read test case files

Read all three files:
- `tasks/compiler_tests/cases2/$ARGUMENTS/case.svelte` — input
- `tasks/compiler_tests/cases2/$ARGUMENTS/case-svelte.js` — expected output (generated by reference Svelte compiler)
- `tasks/compiler_tests/cases2/$ARGUMENTS/case-rust.js` — actual output from our compiler (may not exist)

## Step 3: Identify Svelte features

From `case.svelte`, list all Svelte features exercised:
- **Runes**: $state, $derived, $effect, $props, $bindable
- **Directives**: bind:, class:, style:, on:, use:, transition:, animate:
- **Blocks**: {#if}, {#each}, {#key}, {@const}, {@html}, {#snippet}, {@render}
- **Other**: components, spread attributes, event handlers, slot, reactive assignments

## Step 4: Run test & report status

```
just test-case $ARGUMENTS
```

Report pass or fail.

## Step 5: Diff analysis (only if failing)

Compare `case-rust.js` vs `case-svelte.js`. For each meaningful difference, classify by layer:

1. **Parser/AST** — node not parsed, wrong span, missing AST node
2. **Analysis** — wrong AnalysisData (content_types, dynamic_nodes, runes, element flags)
3. **Transform** — expression not rewritten (rune get/set, prop thunk, mutation wrapping)
4. **Codegen** — wrong JS output, missing runtime call, wrong variable names, ordering issues

## Step 6: Show code path

Using the navigation table from CLAUDE.md, show which files/functions are involved in processing each feature from Step 3:

- **Parser**: which parsing functions handle this feature
- **Analyze**: which visitor passes process it
- **Codegen**: which codegen functions emit the output

## Step 7: Structured output

Format the response as:

```
## Test: $ARGUMENTS

### Features
- [list of Svelte features exercised]

### Status: ✅ pass / ❌ fail

### Diff analysis (if failing)
For each mismatch:
- **What**: [diff description]
- **Layer**: [parser/analyze/transform/codegen]
- **Root cause**: [brief explanation]

### Code path
- Parser: [file:function]
- Analyze: [file:function]
- Codegen: [file:function]

### Suggested next steps
- [what to look at or fix, or "None — test passes" if passing]
```

## Rules

- **Read-only** — NEVER edit any files
- Do not fix the compiler — only explain
- If the case does not exist, report and stop
- If `case-rust.js` does not exist, note that the test has not been run yet and run it
